TITLE: ReScript Advanced Type Definitions and External Bindings
DESCRIPTION: This ReScript code snippet demonstrates how to define opaque types for `school` and `student`, create a `payload` record with a `student` field, and use `@module external` and `@send external` to bind to external JavaScript functions like `getStudentById`. It also shows a conditional logic for querying a student.
SOURCE: https://rescript-lang.org/docs/manual/v12.0.0/introduction/manual/latest/converting-from-js

LANGUAGE: ReScript
CODE:

```
type school
type student
type payload = {
  student: student
}

@module external school: school = "school"
@send external getStudentById: (school, int) => student = "getStudentById"

let defaultId = 10

let queryResult = (usePayload, payload) => {
  if usePayload {
    payload.student
  } else {
    school->getStudentById(defaultId)
  }
}
```

TITLE: Rescript Int.toFixed Function for Fixed-Point String Conversion
DESCRIPTION: The `Int.toFixed` function converts an integer to a string using fixed-point notation. It allows specifying the number of digits after the decimal point. It mirrors JavaScript's `Number.toFixed` and throws a `RangeError` if `digits` is outside the 0-100 range.
SOURCE: https://rescript-lang.org/docs/manual/v12.0.0/introduction/manual/v12.0.0/api/stdlib/int

LANGUAGE: APIDOC
CODE:

```
toFixed:
  Signature: (int, ~digits: int=?) => string
  Description: return a string representing the given value using fixed-point notation. digits specifies how many digits should appear after the decimal point.
  Exceptions:
    RangeError: If digits is less than 0 or larger than 100.
```

LANGUAGE: RESCRIPT
CODE:

```

```

TITLE: Unwrap Rescript Option with getOrThrow
DESCRIPTION: The `getOrThrow` function safely unwraps an `option<'a>` value. If the option is `Some(value)`, it returns `value`. If it's `None`, it throws an exception, optionally with a custom message. This function is useful for scenarios where a value is expected to be present, and its absence indicates an error condition.
SOURCE: https://rescript-lang.org/docs/manual/v12.0.0/introduction/manual/v12.0.0/api/stdlib/option

LANGUAGE: APIDOC
CODE:

```
getOrThrow: (option<'a>, ~message: string=?) => 'a
  Description: Returns 'value' if 'opt' is 'Some(value)', otherwise raises an exception with the message provided, or a generic message if no message was provided.
  Parameters:
    opt: option<'a> - The optional value to unwrap.
    message: string=? - Optional message to use for the exception if 'opt' is 'None'.
  Returns: 'a - The unwrapped value.
  Throws: Error - If 'opt' is 'None'.
```

LANGUAGE: RESCRIPT
CODE:

```
Option.getOrThrow(Some(3))->assertEqual(3)

switch Option.getOrThrow(None) {
| exception _ => assert(true)
| _ => assert(false)
}

switch Option.getOrThrow(None, ~message="was None!") {
| exception _ => assert(true) // Raises an Error with the message "was None!"
| _ => assert(false)
}
```

LANGUAGE: ReScript
CODE:

```

  name: string,

  // mandatory, but explicitly typed as option<int>
  age: option<int>,
```
